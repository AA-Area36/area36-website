## Task 3: Fix Date Display Bug

**File:** `app/grapevine/subscription-drive-section.tsx`

The bug: `new Date("2026-01-31")` parses as UTC midnight, which displays as January 30 in US timezones.

**Fix the `formatDate` function (lines 17-24):**

```tsx
const formatDate = (dateString: string) => {
  // Parse as local date by appending T00:00:00 (avoids UTC interpretation)
  const date = new Date(dateString + "T00:00:00")
  return date.toLocaleDateString("en-US", {
    month: "long",
    day: "numeric",
    year: "numeric",
  })
}
```
